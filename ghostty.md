# Ghostty

## 方針

- Ghosttyをメインターミナルとして使う
- プロジェクトごとにタブを分ける
- タブ内の作業分割はGhosttyのスプリットを使う
- クイックターミナルは一時的な確認や軽いコマンド実行に使う

## チートシート

### 基本

| 操作 | キー |
| --- | --- |
| 設定を開く | `cmd+,` |
| 設定を再読み込み | `cmd+shift+,` |
| 新しいタブ | `cmd+t` |
| 前のタブ | `ctrl+shift+tab` |
| 次のタブ | `ctrl+tab` |
| クイックターミナルを表示/非表示 | `cmd+backquote` |

### スプリット

| 操作 | キー |
| --- | --- |
| 左にスプリット作成 | `cmd+ctrl+h` |
| 下にスプリット作成 | `cmd+ctrl+j` |
| 上にスプリット作成 | `cmd+ctrl+k` |
| 右にスプリット作成 | `cmd+ctrl+l` |
| 左のスプリットへ移動 | `cmd+shift+h` |
| 下のスプリットへ移動 | `cmd+shift+j` |
| 上のスプリットへ移動 | `cmd+shift+k` |
| 右のスプリットへ移動 | `cmd+shift+l` |
| スプリットを左へリサイズ | `cmd+alt+h` |
| スプリットを下へリサイズ | `cmd+alt+j` |
| スプリットを上へリサイズ | `cmd+alt+k` |
| スプリットを右へリサイズ | `cmd+alt+l` |
| 現在のスプリットをズーム | `cmd+shift+enter` |
| スプリット幅を均等化 | `cmd+alt+=` |
| 現在のスプリット/タブ/ウィンドウを閉じる | `cmd+ctrl+w` |
| 閉じたスプリット/タブ/ウィンドウを戻す | `cmd+z` |
| redo | `cmd+shift+z` |

### スクロールとログ確認

| 操作 | キー |
| --- | --- |
| 前のプロンプトへ移動 | `cmd+shift+[` |
| 次のプロンプトへ移動 | `cmd+shift+]` |
| 検索開始 | `cmd+f` |
| 次の検索結果 | `cmd+g` |
| 前の検索結果 | `cmd+shift+g` |

プロンプト移動は、`git status` や `npm test` などのコマンド開始位置へジャンプする操作。長いテスト出力やログから前のコマンド位置へ戻る時に使う。

### クリップボード

| 操作 | 挙動 |
| --- | --- |
| テキスト選択 | 自動コピー |
| 右クリック | コンテキストメニュー |
| 複数行の危険そうな貼り付け | 確認あり |
| bracketed paste対応アプリへの貼り付け | 安全扱い |

## 再起動後のチュートリアル

### 1. Ghosttyを完全終了して起動する

`cmd+q` でGhosttyを終了し、再度Ghosttyを起動する。

確認すること:
- テーマが `GitHub Dark Default` になっている
- フォントが `Menlo`、サイズが `14` になっている
- 背景が透過していない

### 2. プロジェクト用タブを作る

任意のリポジトリへ移動する。

```sh
cd ~/src/github.com/tanaka-takuto/mac_config
```

`cmd+t` で新しいタブを開く。

確認すること:
- 新しいタブも同じ作業ディレクトリから始まる
- プロジェクトごとにタブを分ける感覚で使える

### 3. スプリットで作業面を増やす

プロジェクトのタブ内で次を試す。

```text
cmd+ctrl+l  右にスプリットを作る
cmd+ctrl+j  下にスプリットを作る
cmd+shift+h 左のスプリットへ移動する
cmd+shift+l 右のスプリットへ移動する
```

確認すること:
- 新しいスプリットも同じ作業ディレクトリから始まる
- H/J/K/L の方向感が自然に使える

### 4. スプリットのサイズを調整する

```text
cmd+alt+h  左方向へ20pxリサイズ
cmd+alt+l  右方向へ20pxリサイズ
cmd+alt+j  下方向へ20pxリサイズ
cmd+alt+k  上方向へ20pxリサイズ
```

サイズが崩れたら `cmd+alt+=` で均等化する。

### 5. 集中したいスプリットをズームする

`cmd+shift+enter` を押す。

確認すること:
- 現在のスプリットだけが大きく表示される
- もう一度押すと元の分割表示へ戻る

### 6. 閉じる操作と復元を試す

`cmd+ctrl+w` で現在のスプリットを閉じる。

`cmd+z` で閉じたスプリットを戻す。

確認すること:
- 実行中プロセスがある場合は確認が出る
- 誤って閉じたスプリットを戻せる

### 7. 長い出力でプロンプト移動を試す

次のように複数のコマンドを実行する。

```sh
git status
ls -la
find . -maxdepth 2 -type f
```

`cmd+shift+[` / `cmd+shift+]` で、前後のコマンド開始位置へ移動できるか確認する。

### 8. クイックターミナルを試す

`cmd+backquote` を押す。

確認すること:
- メニューバーのある画面の上から表示される
- 画面の高さ60%で表示される
- 別ウィンドウへフォーカスを移すと自動で隠れる
- もう一度 `cmd+backquote` で呼び出せる

### 9. 貼り付け保護を確認する

改行を含むコマンドを貼り付ける。

```sh
echo one
echo two
```

確認すること:
- 危険そうな貼り付けとして確認が出る
- editorやshellがbracketed paste対応の場合は自然に貼り付けできる

## うまく動かない時

- `cmd+backquote` が反応しない場合は、GhosttyにmacOSのアクセシビリティ権限があるか確認する
- keybindが反映されない場合はGhosttyを完全終了して再起動する
- 作業ディレクトリ継承やプロンプト移動が効かない場合は、shell integrationが有効か確認する
- 設定の構文確認は `ghostty +validate-config` を実行する
